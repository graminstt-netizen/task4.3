# Ассоциативный массив на основе хеш-таблицы (Подсчет частоты слов)

Проект выполнен в рамках лабораторной работы по курсу программирования на языке C.
Автор: Бабурин Дмитрий Сергеевич, группа МК-101.

===================================================================
ОПИСАНИЕ ПРОЕКТА И СТРУКТУРА
===================================================================
Консольное приложение на языке C, реализующее ассоциативный массив на основе хеш-таблицы с разрешением коллизий методом цепочек (односвязные списки).

Особенности реализации:
* Ключами являются строки (слова из текстового файла).
* В узле хранится минимально необходимый набор данных:
  - Указатель на динамически выделенную строку-ключ (`char* key`).
  - Полезные данные — счетчик вхождений (`unsigned int count`).
  - Метаданные — указатель на следующий узел в цепочке коллизий (`struct Node* next`).
* Функционал хеш-таблицы включает:
  - Инициализацию таблицы заданного начального размера.
  - Хеш-функцию (например, MurmurHash, DJB2 или FNV-1a) для равномерного распределения ключей.
  - Добавление элемента (с инкрементом счетчика, если ключ уже существует).
  - Поиск узла по ключу.
  - Удаление узла по ключу.
  - Перестроение (рехеширование) таблицы при превышении коэффициента заполнения (load factor > 0.75) для сохранения высокой скорости поиска O(1).
  - Полную очистку памяти (удаление таблицы).
* Разбор текстового файла: программа считывает файл по словам, приводит их к нижнему регистру, очищает от знаков препинания и заносит в хеш-таблицу.

Структура проекта:
* app/ — исполняемое приложение. Считывает путь к файлу из аргументов командной строки, запускает парсинг, выводит статистику работы таблицы и наиболее частые слова.
* lib/ — библиотека функций. Содержит структуру хеш-таблицы, алгоритм хеширования, операции добавления/поиска/удаления/рехеширования, а также логику разбиения текста на слова.
* tests/ — unit-тесты (минимум 10 тестов) для проверки корректности работы функций библиотеки через фреймворк тестирования (например, стандартный CTest или простейший assert-движок).

ПОДРОБНАЯ ИНСТРУКЦИЯ ПО СБОРКЕ И ЗАПУСКУ В POWERSHELL

1. Очистка кэша и сборка проекта

# Удаляем старую папку сборки (если она была), чтобы избежать конфликтов кэша:
> Remove-Item -Recurse -Force build -ErrorAction SilentlyContinue

# Создаем чистую папку для сборки и переходим в нее:
> mkdir build
> cd build

# Настраиваем CMake (конфигурируем проект с поддержкой тестов):
> cmake ..

# Запускаем компиляцию проекта в конфигурации Debug:
> cmake --build . --config Debug


2. Ручной запуск программы и тестирование

Исполняемый файл `hash_table_app.exe` (или аналогичный по названию вашего таргета) будет находиться в папке `build/app/Debug/`.
Программа принимает путь к текстовому файлу в качестве аргумента командной строки.

* Шаг 3.1. Генерация тестового файла (если нет готового миллионника):
Можно использовать любой текстовый файл (например, скачать книгу в формате .txt) или сгенерировать программно текстовый файл с большим количеством слов.

* Шаг 3.2. Пример запуска анализатора слов:
> ./app/Debug/hash_table_app.exe input.txt
